# AGENTS.md — PrismLabs / PrismOS landing page

Guide for any AI agent or developer working on this repo. Read this first. It lives on every branch.

## What this is
A single-page investor landing site for **PrismLabs**, building **PrismOS** — "AR Ambient OS for physical space." One overhead device reads a room in 3D and casts a live interface ("projected reality") onto the surfaces and objects around you: a wall, a table, a page. No headset, no glasses, no screen. The founders shipped Apple Vision Pro and AR glasses; a prior projected-reality / object-tracking system won a national gold medal, recognized by the U.S. Senate.

Owner: Jonny (jonatechtv@gmail.com). Repo: https://github.com/Jonnysol/prismlabs-landing
Live: **https://meetprism.co** (GitHub Pages + custom domain). `prismlabs.pro` redirects to it. Contact: `hi@meetprism.co`.

## Design system (current — do NOT drift, do NOT revert)
- **Concept:** a dim room lit only by the projection. Dark "viewfinder" staging, real prototype footage full-bleed, deck-tight copy. Reference vibe: Anduril / Linear — serious hardware company.
- **HARD NO #1:** the original cream + Instrument Serif + terracotta look (archived as v1/v2) was rejected as "AI slop." Never go back to it.
- **HARD NO #2:** no rainbow / spectrum gradient text. It was explicitly killed in v13. The spectrum survives ONLY inside the prism logo (the `#spec` gradient). Everything else uses ONE accent.
- **Palette (CSS vars in `:root`):** `--room:#08070A` bg · `--paper:#ECEAF2` text · `--mute:#8E8AA0` · `--accent:#9DA3FF` (one periwinkle accent — used for every emphasis: `em`, highlights, dots, the spatial-computing HUD line).
- **Type:** `--font-display:'Geist'` · `--font-body:'Inter'` · `--font-mono:'Space Mono'` (labels / eyebrows / telemetry). Loaded via Google Fonts. The type scale + weights are CSS vars so the Studio can drive them live.
- **Structure (a deck, not a website):** hero (video-first) → "projected reality" split (text + spinning room scan) → measure reel → proof grid (6 real clips) → "the room reacts" perception band → ambient-at-home → "any room" scene renders → why-now → team / ask → closing flipbook → footer. Keep it tight; cut, don't pad.
- **NO section deletions** without the owner's sign-off ("no deletion allowed" — find a way to make it work instead).
- **Signature:** real footage + the device's own telemetry as the only decoration (mono labels), subtle viewfinder corner ticks. Hero HUD line is "The new generation of spatial computing."

## Copy rules (enforced — the owner cares a lot)
Follow **stop-slop** (https://github.com/hardikpandya/stop-slop):
- **No em dashes. Ever.** Use periods, commas, colons, or restructure.
- No "Wh-" sentence openers in prose, no "not X but Y" binaries, no jargon, no adverb pileups, active voice.
- Sentence case. Short and scannable — no big paragraphs.
- Say **"surfaces and objects"** (not just "surfaces"). Frame the mechanism as **"projected reality"**, never "projector / 3D ceiling."
- Voice: confident, plain, real. Tone lines: "AR Ambient OS for physical space", "only UI appears, black stays invisible", "your home already has walls, we gave it a mind."

## Video rules
- Mobile won't autoplay many videos. Only the hero has `autoplay`; all others are played/paused by an IntersectionObserver (in `index.html`) when they enter/leave the viewport, plus a gesture "kick" fallback. **Keep this pattern.**
- All clips: `muted loop playsinline preload="metadata"` + poster. Transcode to ~720p CRF 30, `-an`, `+faststart`.
- Source footage on the owner's machine (`~/Downloads/...`). `measure.mp4` is the busy/retired one; prefer `measure-2` / `measure-table`.

## OG / link previews
- `index.html` and `admin.html` both carry full Open Graph + Twitter tags with **absolute** URLs (`https://meetprism.co/...`). The preview image is `assets/og.jpg` (1200×630 branded card).
- To regenerate `og.jpg`: build a 1200×630 HTML card (dark, prism logo, tagline, accent), render headless — `Google Chrome --headless=new --window-size=1200,630 --force-device-scale-factor=2 --screenshot=out.png file://card.html` — then downscale with `ffmpeg -vf scale=1200:630 -q:v 3 assets/og.jpg`.
- A page with no OG tags will NOT unfurl in iMessage / Slack. Add tags to any new shareable page.

- **Deploy flow:** commit on a working branch → `git push origin <branch>` → `git checkout main && git merge <branch> --ff-only && git push origin main` → Pages rebuilds (~1 min).
- **Versioning rule (owner preference):** small text / copy / meta tweaks STAY in the same version — do NOT run `build_versions.py` or bump a tag for those. Only run `python3 build_versions.py` for a substantive new design version.

## Studio + version portal
- **Studio** (`studio.js`): visit any page with `?studio` (persists in localStorage). Live-edit fonts, type scale, colours, and inline text. "Copy settings" exports a `:root{…}` block to bake back into `index.html`. Gated so investors never see it.
- **Admin** (`admin.html` at `/admin`): reads `versions.json`, shows every version with a live preview, a favourite picker, a Studio on/off toggle, and an export button. It reflects the versions captured by `build_versions.py`, not raw git history.
